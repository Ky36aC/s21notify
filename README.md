# s21notify

Уведомления **Школы 21** в Telegram: записи на проверку, напоминания, дедлайны — даже когда письма от платформы не доходят.

Маленький демон опрашивает platform.21-school.ru раз в минуту и присылает боту:

- 🔔 **новая запись на проверку** — кто, кого, какой проект, когда;
- 🔁 / ❌ **перенос или отмена** проверки;
- ⏰ **напоминание** за N минут до слота (по умолчанию 30);
- 📅 **дедлайны и экзамены** — появление нового и напоминание за сутки;
- 🏫 **лента уведомлений платформы** («колокольчик»).

И отвечает на команды: `/reviews` — ближайшие проверки, `/agenda` — события на неделю, `/deadlines`, `/status` (уровень, печеньки, коины), `/check` — проверить прямо сейчас.

Вся настройка — через веб-страницу, конфиги руками писать не нужно.

## Быстрый старт

Сначала создай бота: напиши [@BotFather](https://t.me/BotFather) → `/newbot` → скопируй токен.

### Linux / LXC / сервер

```sh
apt install -y git python3-venv        # если ещё нет
git clone https://github.com/Ky36aC/s21notify /opt/s21notify
/opt/s21notify/install.sh
```

Открой `http://<адрес-сервера>:8021`, заполни форму (логин/пароль платформы, токен бота), нажми «Проверить платформу» и «Проверить бота», затем напиши своему боту `/start` — чат привяжется сам. Всё.

### Windows / обычный ПК

1. Скачай проект: `git clone https://github.com/Ky36aC/s21notify` (или Code → Download ZIP).
2. Запусти `start-windows.bat` — он сам создаст окружение и откроет страницу настройки.
3. Заполни форму, напиши боту `/start`.

Автозапуск при входе в Windows: `Win+R` → `shell:startup` → положи туда ярлык на `start-windows.bat`.

## Как это работает

Демон логинится на платформу твоей учёткой (эмуляция входа через Keycloak) и дергает тот же GraphQL API, что и сам сайт. Новые события вычисляются сравнением с прошлым опросом (`state.json`), уведомления шлются через Telegram Bot API. Три потока в одном процессе: опрос платформы, long-polling бота, веб-интерфейс.

## Важно знать

- **API неофициальный.** Шлюз платформы принимает только запросы из белого списка официального клиента — тексты запросов в `s21notify/queries.py` взяты дословно из [s21toolkit/s21schema](https://github.com/s21toolkit/s21schema) (MIT) и менять их нельзя. При обновлении платформы что-то может отвалиться — PR приветствуются.
- **Пароль от платформы хранится локально** в `config.json` открытым текстом (он нужен для перелогина при истечении токена). Файл в `.gitignore` и никуда не отправляется, кроме `auth.21-school.ru`.
- **Веб-интерфейс без пароля** — держи порт 8021 внутри доверенной локальной сети, наружу не пробрасывай.
- К боту может написать `/start` только первый — остальные чаты игнорируются. Отвязать чат можно на странице настроек.
